# Route Catalog 100+100+Small Hundred Peaks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

**Goal:** Publish a source-reviewed HikeSafe catalog with 100 traditional Hundred Peaks, 100 suburban routes, and the Education Ministry Sports Administration's 100 Taiwan Small Hundred Peak designations.

**Architecture:** Keep kind for existing Hundred Peak/suburban behavior. Add a designations JSON array, retain official height difference separately from cumulative ascent, and allow an empty evacuation list only when the authority does not publish one. Apply one additive migration before importing source-reviewed records.

**Tech Stack:** Node.js 24.x, TypeScript, Zod, Vitest, Drizzle ORM, PostgreSQL, Next.js.

## Global Constraints

- Use Node 24.x and npm.
- Keep existing migration files immutable; add drizzle/0008_route_catalog_designations.sql.
- Use official sources only. Never derive ascent from KML, substitute height difference for ascent, or match evacuation points by GIS proximity.
- Retain kind as hundred_peak or suburban. Use only taiwan_small_hundred_peak:001 through taiwan_small_hundred_peak:100 in designations in this scope.
- An empty evacuation list renders 官方資料未載明 and is never a safety claim.
- Do not import into Supabase until npm run routes:verify is valid.
